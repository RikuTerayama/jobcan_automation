# コンテンツ戦略ロードマップ

本ドキュメントは、Jobcan AutoFillの価値を訴求しつつGoogle AdSenseの「有用性が低いコンテンツ」評価を回避するための中長期コンテンツ戦略をまとめたものです。

## 目標

- 6か月以内に専門性・権威性・信頼性（E-E-A-T）を示すコンテンツを20本以上蓄積
- 自然検索からのセッションを月間3,000UUまで拡大
- AdSense審査で重視される「独自性」「有益性」「ユーザー課題の解決度」を満たす

## ガイドライン

1. **一次情報の活用**
   - 導入企業へのインタビュー記事（月1本）
   - 運用データを集計したインサイトレポート（四半期1本）
2. **専門性の明示**
   - 執筆者プロフィール（資格・経歴）を記事末に掲載
   - 引用・参照元を明記し、外部リンクは信頼できる情報源に限定
3. **更新頻度**
   - 月4本（週1本）を最低ラインに設定
   - カテゴリ別の比率：導入ガイド40%、ケーススタディ30%、テクノロジー20%、インサイト10%

## コンテンツパイプライン

| 期間 | 施策 | 目的 |
|------|------|------|
| 1か月目 | ブログカテゴリ整備・初回記事公開 | 情報アーキテクチャの整備 |
| 2か月目 | ケーススタディ追加（業種別×3本） | 導入実績の明示と独自性強化 |
| 3か月目 | 技術解説シリーズ開始（Playwright、セキュリティ） | エンジニア読者の獲得 |
| 4か月目 | ホワイトペーパー「勤怠自動化のROI」公開 | ダウンロード施策でリード獲得 |
| 5か月目 | 動画コンテンツ（ウェビナー・デモ）制作 | マルチメディア化による滞在時間増加 |
| 6か月目 | インサイトレポート（業界別勤怠DX状況） | 被リンク・PR獲得 |

## 記事テンプレート（要約）

1. **導入ガイド記事**
   - 導入背景 → 課題 → 解決策 → 実装手順 → 数値成果 → テンプレート配布
2. **ケーススタディ記事**
   - 企業プロフィール → 課題 → 実施内容 → KPI変化 → 関係者の声
3. **テクノロジー記事**
   - 技術概要 → 設定手順 → ベストプラクティス → トラブル対応 → リファレンス

## 計測・改善

- Search Console：流入キーワードとCTRのモニタリング（週次）
- GA4：滞在時間・スクロール率・コンバージョン率の確認（週次）
- Looker Studioでダッシュボード化し、月次レポートにまとめる

## リソース計画

- ライター：1〜2名（社内＋外部）
- デザイナー：月10時間（図表・OGP画像）
- 動画編集：四半期20時間（ウェビナー編集）
