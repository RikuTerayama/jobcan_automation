📖 勤怠管理・Jobcan用語集

勤怠管理とJobcan AutoFillに関連する用語を分かりやすく解説します。

勤怠管理の基本用語

勤怠管理（きんたいかんり）
Attendance Management
従業員の出勤・退勤時刻、労働時間、休暇取得などを記録・管理することです。本ツールでは、Jobcanへの勤怠データ入力プロセスを自動化することで、勤怠管理の効率化を支援します。Excelファイルに記入した勤怠データを、Jobcanに自動的に入力することで、手作業の時間を大幅に削減できます。
打刻（だこく）
Time Stamping
出勤・退勤の時刻を記録することです。本ツールでは、Jobcanの打刻修正ページにアクセスし、開始時刻と終了時刻を入力して打刻ボタンをクリックする操作を自動化します。人間の操作パターンを再現するため、各データ処理後に適切な待機時間を設けています。
例：本ツールでは、「データ 1/11: 2025-11-13 09:00:00-22:30:00」のように、日付と時刻を自動的に打刻します。
開始時刻（かいしじこく）
Start Time
本ツールでは、ExcelファイルのB列に入力する勤務開始時刻を指します。「09:00」または「0900」形式で入力してください。本ツールは、この時刻をJobcanの打刻修正ページに入力し、1回目の打刻ボタンをクリックします。
例：ExcelファイルのB列に「09:00」と入力すると、本ツールは「⏰ 1回目: 始業時刻を入力: 0900」というログを出力し、Jobcanに「0900」と入力します。
終了時刻（しゅうりょうじこく）
End Time
本ツールでは、ExcelファイルのC列に入力する勤務終了時刻を指します。「18:00」または「1800」形式で入力してください。本ツールは、この時刻をJobcanの打刻修正ページに入力し、2回目の打刻ボタンをクリックします。
例：ExcelファイルのC列に「22:30」と入力すると、本ツールは「⏰ 2回目: 終了時刻を入力: 2230」というログを出力し、Jobcanに「2230」と入力します。

Jobcan関連用語

Jobcan（ジョブカン）
Jobcan
株式会社DONUTSが提供する勤怠管理システムです。Web上で出退勤の打刻、勤怠データの管理、シフト作成などが行えます。多くの企業で導入されています。
勤怠データ（きんたいデータ）
Attendance Data
出勤時刻、退勤時刻、労働時間、休暇取得状況などの記録データです。給与計算や労務管理に使用されます。
打刻修正（だこくしゅうせい）
Time Stamp Correction
間違って打刻した時刻や、打刻忘れを後から修正することです。本ツールでは、Jobcanの打刻修正ページ（https://ssl.jobcan.jp/employee/adit/modify?year=YYYY&month=MM&day=DD）にアクセスし、開始時刻と終了時刻を入力して打刻ボタンをクリックする操作を自動化します。各日付ごとに打刻修正ページに移動し、時刻を入力して打刻ボタンをクリックする処理を繰り返します。
例：本ツールは、「🔗 打刻修正ページに移動: https://ssl.jobcan.jp/employee/adit/modify?year=2025&month=11&day=13」というログを出力し、該当ページにアクセスします。

Jobcan AutoFill関連用語

自動入力（じどうにゅうりょく）
Automatic Data Entry
人間が手動で行っていた入力作業を、プログラムが自動的に実行することです。本サービスの中核機能です。
テンプレートファイル
Template File
あらかじめ日付が入力されたExcelファイルのことです。開始時刻と終了時刻を入力するだけで使用できるため、手間を大幅に削減できます。
Excel形式（エクセルけいしき）
Excel Format
Microsoft Excelで作成・編集できるファイル形式です。本サービスでは.xlsx または .xls形式に対応しています。
アップロード
Upload
ローカルコンピュータのファイルをサーバーに送信することです。本サービスでは、勤怠データを記入したExcelファイルをアップロードして処理を実行します。
処理状況（しょりじょうきょう）
Processing Status
自動入力の進行状況を示す情報です。本ツールでは、ポップアップウィンドウにリアルタイムで処理状況が表示されます。具体的には、以下のような情報が表示されます：
  • 「📝 データ X/11: YYYY-MM-DD HH:MM-HH:MM」：現在処理中のデータ
  • 「✅ データ X/11 の処理が完了しました」：各データの処理完了
  • 「📊 全データ処理完了: X件のデータを処理しました」：全データの処理完了
  • 「🎉 処理が正常に完了しました」：最終的な処理完了
エラーが発生した場合は、エラーメッセージとその詳細が表示されます。
ログイン認証（ログインにんしょう）
Login Authentication
ユーザーの身元を確認するプロセスです。メールアドレスとパスワードを使用して、正当なユーザーであることを証明します。

IT・セキュリティ用語

HTTPS（エイチティーティーピーエス）
HyperText Transfer Protocol Secure
暗号化されたHTTP通信のことです。データの盗聴や改ざんを防ぎ、安全に情報をやり取りできます。本サービスではHTTPSを使用しています。
画像認証（がぞうにんしょう）
CAPTCHA
自動プログラム（ボット）ではなく、人間が操作していることを確認するための仕組みです。本ツールでは、CAPTCHAが表示された場合、処理を中断します。これは、セキュリティ上の理由からです。CAPTCHAが表示された場合は、一度手動でJobcanにログインし、CAPTCHAをクリアしてから、30分〜1時間程度待ってから再度処理を試行してください。
例：短時間に複数回ログインを試みると、Jobcan側のセキュリティチェックが発動し、CAPTCHAが表示される場合があります。
Cookie（クッキー）
Cookie
Webサイトがブラウザに保存する小さなデータファイルです。ログイン状態の保持や、ユーザーの設定を記憶するために使用されます。
キャッシュ
Cache
一度アクセスしたデータを一時的に保存しておく仕組みです。次回アクセス時に高速に表示できますが、古いデータが表示される場合もあります。
タイムアウト
Timeout
処理が一定時間内に完了しなかった場合、自動的に中断されることです。ネットワーク接続が不安定な場合などに発生します。
Playwright（プレイライト）
Playwright
本ツールで使用しているブラウザ自動化ライブラリです。Playwrightを使用することで、実際のブラウザを操作し、人間の操作パターンを再現します。具体的には、Jobcanのログインページにアクセスし、メールアドレスとパスワードを人間らしいタイピング速度で入力し、ログインボタンをクリックする操作を自動化します。
例：本ツールは、Playwrightを使用して「⌨️ 人間らしいタイピングを実行 (試行 1/3): input[name="user[email]"]」というログを出力し、人間らしい操作を再現します。
処理間隔（しょりかんかく）
Processing Interval
本ツールでは、各データ処理後に設けられる待機時間を指します。1〜3件目は約2秒、4件目以降は約5秒の待機時間が設けられています。この待機時間は、Jobcan側のサーバー負荷を軽減し、セキュリティチェックが発動しにくくするために設けられています。
例：4件目以降のデータ処理では、「⏳ メモリ最適化のため5秒待機中...」というログが表示され、5秒間待機します。
空白行のスキップ（くうはくぎょうのスキップ）
Blank Row Skipping
本ツールでは、Excelファイル内の空白行を自動的にスキップする機能です。日付、開始時刻、終了時刻がすべて空白の行は、処理対象から除外されます。これにより、休日や欠勤の日を空白行として残しておくことができます。
例：Excelファイルに11行のデータがあり、そのうち2行が空白の場合、本ツールは「✅ Excelファイル読み込み完了: 11件のデータ」「✅ データ処理で空白行 2 行をスキップしました」「✅ Excelファイル読み込み完了: 9件の有効データ」というログを出力します。

就業規則とコンプライアンス

就業規則（しゅうぎょうきそく）
Work Rules / Employment Regulations
企業が定める労働条件や職場のルールです。本ツールを使用する際は、所属企業の就業規則やセキュリティポリシーを必ず遵守してください。自動化ツールの使用が禁止されている場合は、使用を控えてください。詳細は利用規約をご確認ください。
